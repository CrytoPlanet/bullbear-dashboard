# BullBear Dashboard 中英双语设计文档

- 日期：2026-05-08
- 作者：Claude Code (与 ivan.jin 协作)
- 状态：已批准 / 待实施

## 目标

把当前仅有中文的前端 dashboard 改造为中英双语，可由用户在 header 右上角的 `[中|EN]` 切换器实时切换。覆盖所有用户可见文案，包括 UI 标签、tooltip 详细说明、四象限策略指导、状态切换信号文案等。后端不动。

## 非目标

明确**不做**的事：

- 代码注释、`console.warn` 等调试输出 —— 保留原状（中文）
- URL 参数同步语言（如 `?lang=en`）—— YAGNI
- 后端任何改动 —— 后端继续返回中文枚举字符串作为内部标识符
- 数字/日期格式本地化 —— 项目已统一使用 `Intl.NumberFormat('en-US')`，两种语言都沿用美式格式
- 新增自动化测试 —— 项目目前没有测试基础设施，不在本次范围内引入
- 第三种语言的扩展（先做好双语，结构上不阻碍未来扩展）

## 架构总览

### 技术选型

- **i18n 库**：`vue-i18n@9`，使用 Composition API 模式（`legacy: false`）
- **文案存储**：TypeScript 文件（非 JSON），便于类型检查和共享类型 schema
- **持久化**：`localStorage`，key 名 `locale`
- **首次默认值**：`navigator.language` 检测，以 `zh` 开头则中文，否则英文

### 文件改动

新增：
```
frontend/src/locales/
├── index.ts        # createI18n 初始化、语言检测、setLocale 暴露
├── zh.ts           # 中文文案字典
└── en.ts           # 英文文案字典
```

修改：
```

frontend/src/main.ts      # +1 行 app.use(i18n)
frontend/src/types.ts     # 删除 DATA_LABELS 和 QUADRANT_MAP（其 cn/en/trend/flow 全部由 i18n 字典提供）
frontend/src/App.vue      # 模板和 script 内所有中文 → t('…')；新增 lang-toggle UI
```

## 关键设计抉择

### 抉择 1：后端枚举值保持中文，不翻译

**问题**：后端返回的 `state` / `trend` / `funding` 是中文字符串（`'牛市进攻'`、`'趋势多'`、`'资金进攻'`），前端代码里被当作条件判断 key 使用（`v-if="state === '牛市进攻'"`）。

**决定**：把它们当作"内部标识符"，**不翻译，不改后端**。仅在显示时通过映射表查 i18n 字典。

```ts
// App.vue 内常量
const STATE_KEYS: Record<string, string> = {
  '牛市进攻': 'bullOffensive',
  '牛市修复': 'bullRecovery',
  '熊市反弹': 'bearBounce',
  '熊市消化': 'bearDigest',
};
const TREND_KEYS: Record<string, string> = {
  '趋势多': 'bull',
  '趋势空': 'bear',
};
const FUNDING_KEYS: Record<string, string> = {
  '资金进攻': 'offensive',
  '资金防守': 'defensive',
  '无法判断': 'unknown',
};
```

显示用 `t('quadrant.' + STATE_KEYS[state] + '.name')`，判断逻辑保持原样不动。

### 抉择 2：script 长文计算函数 → 响应式 computed

**问题**：`getMaPriceRelation()`、`getTrendStructure()`、`getFundingPattern()`、`calculateTransitionSignals()` 等函数返回中文字符串对象（`{statusText, summary, name, description}`）。

**决定**：把它们重写为响应式 `computed`，内部直接调 `t()` 拼装：

```ts
const maPriceRelation = computed(() => {
  if (!data.value.btc_price || !data.value.ma50 || !data.value.ma200) {
    return { statusText: t('maPrice.unknown.status'), summary: t('maPrice.unknown.summary') };
  }
  // 逻辑判断 → 选定 key
  return {
    statusText: t('maPrice.aboveBoth.status'),
    summary: t('maPrice.aboveBoth.summary'),
  };
});
```

**理由**：
- `t` 是 vue-i18n 提供的响应式函数，locale 变化时 computed 自动重算
- 这些函数当前已经包在 `computed()` 里，重写代价最小
- 模板侧几乎无需改动

被否的备选：函数返回 key 字符串、模板里再 `t()`。会让模板新增大量 `t(...)` 调用，且对组合返回的对象（如 `{statusText, summary}`）需要在两处分别 `t()`，更繁琐。

## 切换器与初始化

### 语言切换器（header 右上角）

```vue
<div class="lang-toggle">
  <button :class="{ active: locale === 'zh' }" @click="setLocale('zh')">中</button>
  <button :class="{ active: locale === 'en' }" @click="setLocale('en')">EN</button>
</div>
```

样式：低调圆角小按钮，黑底白字，`active` 态高亮。绝对定位 / flex 末尾对齐到 header 右上角。

### `setLocale(loc)` 三步

```ts
function setLocale(loc: 'zh' | 'en') {
  i18n.global.locale.value = loc;
  localStorage.setItem('locale', loc);
  document.documentElement.lang = loc === 'zh' ? 'zh-CN' : 'en-US';
}
```

### 初始化（`locales/index.ts`）

```ts
function detectInitialLocale(): 'zh' | 'en' {
  const saved = localStorage.getItem('locale');
  if (saved === 'zh' || saved === 'en') return saved;
  return navigator.language.toLowerCase().startsWith('zh') ? 'zh' : 'en';
}

const initial = detectInitialLocale();
const i18n = createI18n<[MessageSchema], 'zh' | 'en'>({
  legacy: false,
  locale: initial,
  fallbackLocale: 'en',
  messages: { zh, en },
});

// 应用初始 <html lang>
document.documentElement.lang = initial === 'zh' ? 'zh-CN' : 'en-US';

export default i18n;
```

## 数据流

1. **应用启动**：`main.ts` `app.use(i18n)` → i18n 在 `detectInitialLocale()` 决定语言
2. **首次渲染**：组件内 `useI18n()` 取 `t`、`locale`；模板调 `t('app.title')`
3. **后端响应**：`stateData.state = '牛市进攻'`（中文标识符）
4. **显示**：模板 `{{ t(\`quadrant.\${STATE_KEYS[stateData.state]}.name\`) }}` → 渲染当前语言文案
5. **用户切换**：点击 EN → `setLocale('en')` → `i18n.global.locale.value` 变 → 所有 `t()` 重新求值 → DOM 重渲染 + `<html lang>` 更新 + localStorage 写入

`currentQuadrant` 现状是 `computed` 从 `QUADRANT_MAP` 取数据；改为：
```ts
const currentQuadrant = computed(() => {
  if (!stateData.value?.ok) return null;
  const key = STATE_KEYS[stateData.value.state];
  if (!key) return null;
  return {
    key,
    name: t(`quadrant.${key}.name`),
    trendLabel: t(`quadrant.${key}.trendLabel`),
    flowLabel: t(`quadrant.${key}.flowLabel`),
  };
});
```

## 错误处理与边界

- 字典 key 缺失：vue-i18n 默认 fallback 到 `'en'`；若英文也缺，渲染 key 字符串本身（开发期可见）
- localStorage 不可用（隐私模式）：`setItem` 静默失败；不影响切换，仅是不持久化
- 后端返回未知 `state` 值（不在 `STATE_KEYS` 表）：映射结果 `undefined` → 显示 key 字符串原文。这是诊断信号，不做隐藏 fallback
- `navigator.language` 缺失（极少见）：fallback 到英文

## 风险与备注

- **翻译质量**：英文文案涉及金融术语（多头排列 / Bullish stacking、空头排列 / Bearish stacking、资金进攻 / Offensive flow 等），实施阶段需要逐条审稿；初稿后可让用户校对一遍
- **App.vue 文件已经 3148 行**：本次不在重构范围；但若发现某些 section 转 i18n 后行数膨胀严重，可在实施过程中把孤立的 section（如 tooltip 内容）拆到子组件 —— 仅当迁移压力切实需要时才做
- **vue-i18n 包大小**：约 +20KB gzip，对静态部署的影响可忽略
